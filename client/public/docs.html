PDF Processing

Endpoints for uploading and analyzing PDF/DOCX files. Supports text extraction, encryption checks, PDF/DOCX processing, and asking natural language questions about the content.

POST /pdf

Extract PDF: Upload a PDF/DOCX file and extract its text or images. Automatically applies PDF/DOCX processing if the document is scanned or encrypted.

Example

curl -X POST "http://localhost:8000/pdf" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@example.pdf" \
  -F "dpi=150" \
  -F "extract_images=true"

Response

{
  "status": "success",
  "text": "Extracted text content ...",
  "results": null,
  "message": null
}
POST /check-pdf

Check PDF: Quickly determine if a PDF is encrypted, requires PDF/DOCX processing, and get a sample of extracted text for verification.

Example

curl -X POST "http://localhost:8000/check-pdf" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@example.pdf"

Response

{
  "isEncrypted": false,
  "requiresPDFProcessing": false,
  "sampleText": "Hello World..."
}
POST /ask-pdf

Ask PDF: Send natural language queries to extract semantic answers from the document. Useful for summaries, Q&A, and searching within content.

Example

curl -X POST "http://localhost:8000/ask-pdf" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@example.pdf" \
  -F "question=What is the summary of this pdf?"

Response

{
  "status": "success",
  "summary": "This is a summary of the PDF content..."
}
POST /message-verify

Message Verify: Ensures requests truly originate from Slack by validating signatures and timestamps.

Example

curl -X POST "http://localhost:8000/message-verify" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello from Slack!", "timestamp": "2024-06-01T12:00:00Z"}'

Response

{
  "status": "verified",
  "message": "Request is authentic."
}
